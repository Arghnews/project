Vulkan SDK for Linux Release Notes

Vulkan SDK 1.0.30.0 Release Notes

Version 1.0.30.0 for Linux
October 13, 2016

This SDK supports Vulkan API revision 1.0.30. The prior SDK supported Vulkan API revision 1.0.26.

NOTE: Please see the Known Driver Issues section in this document for information about a compatibility issue between glslang and Nvidia drivers.

Please refer to the Getting Started Guide for a list of supported Linux distributions.

Device layers are now deprecated! Any device layers must be converted to a layer that is queried and enabled at instance level. vkEnumerateDeviceLayerProperties is deprecated.

Linux SDK now installs itself into a local directory tree! The installer no longer uses debian packages that install files into system directories. See Getting Started Guide for more information.

Overview of new features in SDK 1.0.30.0

  • The LunarG Samples Progression now includes a Vulkan Tutorial to accompany the sample program code
  • A new Vulkan Sample has been added that demonstrates the use of Vulkan Input Attachments
  • The Unique Objects utility layer has been updated and improved
  • A new Cube.cpp sample demonstrates the use of the vulkan.hpp low-level Vulkan C++ API
  • Improved coverage and utility in layer validation tests
  • Scores of bugfixes, new validation checks, and feature additions
  • The tri.exe demonstration app has been removed

This version of the SDK components are based on the following specifications and source code repositories:

  • Vulkan Specification including extensions

  • Vulkan Loader and Validation version 1.0.30 of vulkan.h

  • SPIR-V Specification version 1.1

  • Glslang library and tools

  • SPIR-V headers

  • SPIR-V tools

  • Vulkan Tools

  • Vulkan Samples

Developer Notes

  • The pre-compiled glslang static libraries are not compatible with the Linux 4 runtime (e.g., Ubuntu 15.10). These libraries must be rebuilt before use with the SDK samples. See the Vulkan Samples page for more information.

Known Driver Issues

  • Update Nvidia Drivers
    • A recent glslang change exposed a bug in the texel fetch behavior on Nvidia devices under certain situations.
    • Nvidia has fixed the issue and posted drivers available with the fix to their download page.
    • Nvidia Linux Drivers version 367.35 and newer should include the fix.
